The Vue package is the base integration for the framework. It exposes the context registration functions, all query and mutation composables, and the higher-level controllers.

## Mental Model

The Vue package is the lowest-level frontend integration for Ginjou.

1. Register providers once near the root.
2. Resolve resources and routing from your Vue application.
3. Build pages with composables first, then controllers when the route fits a standard CRUD shape.

If you are not using Nuxt, this is the integration to start from.

## Setup

Ginjou is wired through root-level context registration. Instead of one monolithic plugin, you explicitly register the capabilities your app uses.

### Common Context Functions

| Function | Purpose |
| :--- | :--- |
| `defineQueryClientContext` | Register the TanStack Query client. |
| `defineFetchersContext` | Register one or more fetchers. |
| `defineResourceContext` | Register resources and route patterns. |
| `defineRouterContext` | Register router integration. |
| `defineAuthContext` | Register authentication. |
| `defineAuthzContext` | Register authorization. |
| `defineNotificationContext` | Register notifications. |
| `defineRealtimeContext` | Register realtime support. |
| `defineI18nContext` | Register translations. |

### Setup Example

::code-group

```vue [Vue]
<script setup lang="ts">
import { defineFetchersContext, defineQueryClientContext, defineResourceContext, defineRouterContext } from '@ginjou/vue'
import { createFetcher } from '@ginjou/with-rest-api'
import { createRouter } from '@ginjou/with-vue-router'
import { QueryClient } from '@tanstack/vue-query'

defineQueryClientContext(new QueryClient())

defineFetchersContext({
	default: createFetcher({
		url: 'https://api.example.com',
	}),
})

defineResourceContext({
	resources: [
		{
			name: 'posts',
			list: '/posts',
			show: '/posts/:id',
			edit: '/posts/:id/edit',
		},
	],
})

defineRouterContext(createRouter())
</script>

<template>
	<RouterView />
</template>
```

### Minimal App Shape

The recommended structure is:

1. `App.vue` registers providers.
2. Resources define route-aware pages.
3. Route components call composables or controllers.

That keeps framework wiring centralized and feature logic local.

## Recommended Supporting Packages

- `@ginjou/with-vue-router` for route integration
- `@ginjou/with-vue-i18n` for i18n integration
- a backend provider such as `@ginjou/with-rest-api`, `@ginjou/with-supabase`, or `@ginjou/with-directus`

## Working Style In Vue

The recommended pattern is:

1. Register providers once in `App.vue`.
2. Use low-level composables for specialized features.
3. Use controllers for standard resource screens.

That keeps application wiring centralized and feature logic local.
