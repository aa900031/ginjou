The Nuxt module adds SSR-friendly integration on top of the Vue package. You still use the same Ginjou contracts, but the module handles router integration and provides async composables for server rendering.

## Mental Model

The Nuxt module keeps the same Ginjou programming model while adapting it to SSR.

1. Providers are still registered at the root app level.
2. Router integration is supplied by the module.
3. Async composables participate in Nuxt server rendering.

Choose this integration when route-level data should be ready during SSR or hydration.

## Setup

### nuxt.config.ts

Register the module first.

```ts [nuxt.config.ts]
export default defineNuxtConfig({
	modules: ['@ginjou/nuxt'],
})
```

### app.vue

Register Ginjou providers in `app.vue` or `app/app.vue`.

::code-group

```vue [Nuxt]
<script setup lang="ts">
import { defineFetchersContext, defineQueryClientContext, defineResourceContext } from '@ginjou/vue'
import { createFetcher } from '@ginjou/with-rest-api'
import { QueryClient } from '@tanstack/vue-query'

defineQueryClientContext(new QueryClient())

defineFetchersContext({
	default: createFetcher({
		url: 'https://api.example.com',
	}),
})

defineResourceContext({
	resources: [
		{
			name: 'posts',
			list: '/posts',
			show: '/posts/:id',
		},
	],
})
</script>

<template>
	<NuxtLayout>
		<NuxtPage />
	</NuxtLayout>
</template>
```

### Minimal App Shape

The recommended structure is:

1. Register the Nuxt module.
2. Register providers in `app.vue`.
3. Use async composables in route-level pages that should render with data on the server.
4. Use standard Vue composables for client-only interactions.

## Async Composables

The Nuxt module provides SSR-ready async wrappers for the main query and controller flows.

### Available Async Composables

- `useAsyncGetOne`
- `useAsyncGetMany`
- `useAsyncGetList`
- `useAsyncGetInfiniteList`
- `useAsyncShow`
- `useAsyncList`
- `useAsyncEdit`
- `useAsyncSelect`
- `useAsyncInfiniteList`
- `useAsyncAuthenticated`
- `useAsyncGetIdentity`
- `useAsyncPermissions`
- `useAsyncCanAccess`

### Example

::code-group

```vue [Nuxt]
<script setup lang="ts">
const { records, isLoading } = await useAsyncGetList({
	resource: 'posts',
})
</script>

<template>
	<div v-if="isLoading">
		Loading...
	</div>
	<ul v-else>
		<li v-for="post in records" :key="post.id">
			{{ post.title }}
		</li>
	</ul>
</template>
```

## Practical Guidance

- Put all provider registration in the root app component.
- Keep registration unconditional to avoid SSR hydration mismatches.
- Use async composables for route-level data that should be ready during server rendering.
- Use the standard Vue composables for purely client-side interactions.
