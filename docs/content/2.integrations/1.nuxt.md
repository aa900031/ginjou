The `@ginjou/nuxt` module provides first-class support for Nuxt 3 or 4, including SSR-ready composables and automatic provider configuration.

## Installation

Install the module and its required peer dependency:

::code-group

```bash [pnpm]
pnpm add @ginjou/nuxt @tanstack/vue-query
```

```bash [yarn]
yarn add @ginjou/nuxt @tanstack/vue-query
```

```bash [npm]
npm install @ginjou/nuxt @tanstack/vue-query
```

```bash [bun]
bun add @ginjou/nuxt @tanstack/vue-query
```

::

## Configuration

Add the module to your `nuxt.config.ts`:

```typescript
export default defineNuxtConfig({
	modules: [
		'@ginjou/nuxt'
	],
	ginjou: {
		// Enable/disable automatic router integration (default: true)
		router: true
	}
})
```

## Async Composables

To fully support Server-Side Rendering (SSR), the module provides "Async" versions of standard composables. these composables automatically handle `onServerPrefetch` and `suspense` to ensure data is fetched on the server before the page is rendered.

Common async composables include:
- **Query**: `useAsyncGetList`, `useAsyncGetOne`, `useAsyncGetMany`
- **Controller**: `useAsyncList`, `useAsyncShow`, `useAsyncEdit`
- **Auth**: `useAsyncAuthenticated`, `useAsyncGetIdentity`

### Usage

```vue
<script setup lang="ts">
// No import needed - auto-imported by Nuxt!
const { data, isLoading } = await useAsyncGetList({
	resource: 'posts'
})
</script>

<template>
	<div v-if="isLoading">
		Loading...
	</div>
	<ul v-else>
		<li v-for="post in data" :key="post.id">
			{{ post.title }}
		</li>
	</ul>
</template>
```

## Automatic Integrations

The Nuxt module minimizes boilerplate by automatically setting up common integrations.

### Vue Router

The framework automatically integrates with `vue-router` via `@ginjou/with-vue-router`. This means `useGo`, `useBack`, and resource resolution work out of the box without any manual provider setup.

### Vue I18n

If you have `@nuxtjs/i18n` installed in your project, the module will automatically detect it and install the `@ginjou/with-vue-i18n` provider. This enables consistent translation support across all framework components.
