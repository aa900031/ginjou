# I18n Reference

Use this reference when the task includes wiring localization, internationalization, translation support, locale switching, or `vue-i18n` into a Ginjou app.

## Provider Rule

I18n is an app-level integration. Register it once with `defineI18nContext` using the `@ginjou/with-vue-i18n` adapter, or supply a custom adapter.

## Setup

```typescript
import { defineI18nContext } from '@ginjou/vue'
import { createI18n } from '@ginjou/with-vue-i18n'
import { createI18n as createVueI18n } from 'vue-i18n'

const vueI18n = createVueI18n({
  locale: 'en',
  messages: {
    en: { greeting: 'Hello', welcome: 'Welcome {name}' },
    zh: { greeting: '你好', welcome: '歡迎 {name}' },
  },
})
defineI18nContext(createI18n(vueI18n))
```

## Provider Contract

```typescript
import { defineI18n } from '@ginjou/core'

// Custom adapter (if not using @ginjou/with-vue-i18n)
defineI18nContext(defineI18n({
  translate: (key, params?) => {
    // return translated string or undefined if key is missing
    return messages[locale][key]
  },
  getLocale: () => locale,
  setLocale: (value) => { locale = value },
  onChangeLocale: (handler) => {
    subscribes.add(handler)
    return () => subscribes.delete(handler) // cleanup fn
  },
}))
```

## Composable Usage

```typescript
import { useLocale, useTranslate } from '@ginjou/vue'

// useTranslate returns the translate function DIRECTLY (not an object)
const t = useTranslate()
t('greeting')            // → 'Hello'
t('welcome', { name: 'Mark' }) // → 'Welcome Mark'
// Note: returns undefined if the key is missing — not an error

// useLocale returns a WRITABLE Ref<string>
const locale = useLocale()
console.log(locale.value) // current locale, e.g. 'en'
locale.value = 'zh'       // switch locale (calls setLocale under the hood)
// Can also be used with v-model in a <select>
```

## Important: `useTranslate` API

`useTranslate()` returns a **function** directly, not an object. Do not destructure it:

```typescript
// CORRECT
const t = useTranslate()
t('key')

// WRONG — will cause a runtime error
const { translate: t } = useTranslate()
```

## `useLocale`

`useLocale()` returns a writable `Ref<string>`. It reads the current locale and updates it when assigned. It **requires** `getLocale`, `setLocale`, and `onChangeLocale` to be implemented in the provider — it throws if any of these is missing.

## Guidance

- `@ginjou/with-vue-i18n` bridges `vue-i18n` to the Ginjou I18n interface.
- Ginjou uses translations internally for default notification messages and resource labels.
- You can register a custom adapter if you prefer a different i18n library — implement the `I18n` interface from `@ginjou/core`.
- `translate` may return `undefined` if a key is missing. Guard or provide fallbacks where an undefined label would cause UI issues.

## Common Mistakes

- Registering i18n conditionally instead of at the app root.
- **Destructuring `useTranslate()`** — it returns a function, not an object.
- Calling `useLocale()` without implementing `getLocale`/`setLocale`/`onChangeLocale` in the provider.
- Forgetting that `translate` may return `undefined` for missing keys.
