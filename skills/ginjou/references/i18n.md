# I18n Reference

Use this reference when adding localization, translation, locale switching, or `vue-i18n` integration to a Ginjou app.

## Concept

Ginjou wraps your i18n library with a standard adapter. Use `@ginjou/with-vue-i18n` for `vue-i18n`, or implement the `I18n` interface from `@ginjou/core` for any other library.

## Setup with `vue-i18n`

```typescript
import { defineI18nContext } from '@ginjou/vue'
import { createI18n } from '@ginjou/with-vue-i18n'
import { createI18n as createVueI18n } from 'vue-i18n'

const vueI18n = createVueI18n({
	locale: 'en-US',
	messages: {
		'en-US': { hi: 'Hello', msg: 'Hello {name}' },
		'zh-TW': { hi: '你好', msg: '您好 {name}' },
	},
})
defineI18nContext(createI18n(vueI18n))
```

## Composable Usage

```vue
<!-- From stories/vue/src/I18nLocale.vue -->
<script setup lang="ts">
import { useLocale, useTranslate } from '@ginjou/vue'

// useTranslate returns a FUNCTION directly — do not destructure
const t = useTranslate()

// useLocale returns a writable Ref<string>
const locale = useLocale()
</script>

<template>
	Locale: {{ locale }}<br>
	<ul>
		<li>'hi' >>> {{ t('hi') }}</li>
		<li>'msg' >>> {{ t('msg', { name: 'Mark' }) }}</li>
	</ul>
	<!-- locale is writable — can use v-model -->
	<select v-model="locale">
		<option value="en-US">
			en-US
		</option>
		<option value="zh-TW">
			zh-TW
		</option>
	</select>
</template>
```

## Provider Contract (Custom Adapter)

```typescript
import { defineI18n } from '@ginjou/core'

defineI18nContext(defineI18n({
	translate: (key, params?) => {
		return messages[locale.value][key] // may return undefined for missing keys
	},
	getLocale: () => locale.value,
	setLocale: (value) => { locale.value = value },
	onChangeLocale: (handler) => {
		watch(locale, handler)
		return () => {} // cleanup function
	},
}))
```

## Rules

- `useTranslate()` returns a function, **not** an object. Do not destructure it.
- `useLocale()` returns a writable `Ref<string>`. It requires `getLocale`, `setLocale`, and `onChangeLocale` to be implemented.
- `translate` may return `undefined` for missing keys — guard against it where an undefined label would cause UI issues.
- Register `defineI18nContext` unconditionally at the app root.
